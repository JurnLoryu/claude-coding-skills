# 对抗性复核（高风险 / 正确性敏感才上）

触发：分支逻辑、跨模块边界、难验证的不变量、不可逆操作。五步：

1. **CLAIM** — 两句话说决策 + 为何重要。
2. **EXTRACT** — 隔离出制品（代码片段/契约），去掉推理过程。
3. **DOUBT** — 找独立审查员，指令必须**对抗性**（"找缺陷"，不是"好不好"）；**只给制品 + 验收契约，不给你的结论**。
4. **RECONCILE** — 按优先级分类：契约误读 > 可行动真问题 > 权衡 > 噪声。
5. **STOP** — 三轮上限 / 发现已微不足道 / 用户明确同意。

外部 CLI：先验 PATH → 测 binary → 确认语法 → 显式授权。
