# 接口 / 契约设计

- **契约优先**：接口先于实现，类型即规范；设计成"做对容易、做错难"。
- **Hyrum 定律**：可观察的行为都会被依赖 → 隐藏实现细节。
- **一致错误语义**：全局一套错误形状（如 `{code,message,details}`），别有的 throw、有的返回 null。
- **边界验证**：只在入口（API/表单/三方响应）验；三方数据**永不信任**，先验再用。
- **向后兼容**：新字段加可选，别改类型/删字段；列表端点必须分页。
- 神器：**判别联合类型 + 类型守卫**（`{type:'pending'} | {type:'in_progress', assignee}`），消费者自动类型收窄，用不错变体。
