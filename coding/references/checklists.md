# 清单

## 实现前
- [ ] 读懂相关代码与所有调用点？
- [ ] 需求清楚？不清楚先问或先写规格（大项目走 spec→plan→tasks）。
- [ ] 大任务已纵向切片、按依赖排序、每个 S/M 级？
- [ ] 关键决策（范围/方案/不可逆）已和用户确认？
- [ ] 框架用法已查官方文档、版本确定？

## 实现中
- [ ] 每步 ≤100 行、只改一件事、改完即编译+测试+验证？
- [ ] 贴合既有风格、没顺手塞无关改动？
- [ ] 逻辑改动/修 bug 有对应测试（bug 先写复现测试）？
- [ ] 边界与失败路径都处理了？

## 提交前
- [ ] 测试/构建跑过，结果**如实**记录（失败贴输出）？
- [ ] 五维过一遍：正确/可读/架构/安全/性能？
- [ ] 能简化的已简化（不改行为）；死代码已确认再删？
- [ ] `git diff --staged` 看过 + grep 密钥；提交原子、信息清楚、非默认分支？
- [ ] 变更摘要 + "没碰清单" + 假设确认项？
- [ ] 非显然的修复/标准/坑已按 memory 技能沉淀？

## 上线前（大项目）
- [ ] 代码/安全/性能三块清单过关？
- [ ] 功能开关就位、灰度计划与阈值定好？
- [ ] 监控（应用/基础/客户端）+ 回滚计划三件套备好？
